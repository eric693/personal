個人管理學苑 / Module 04

工作管理
效能系統建立

忙是一種感覺，完成才是事實。許多人每天從早忙到晚，卻在夜晚回顧時發現真正重要的事沒有推進。工作管理不是讓你做更多事，而是確保你在做對的事——建立一套從任務捕捉、整理、執行到收尾的可靠系統，讓大腦從記憶的重擔中解放出來。

核心觀念框架

Concept 01
外部化思考

大腦是用來思考的，不是用來記憶的。把所有任務、想法、承諾移出大腦，放進可信賴的系統。

Concept 02
下一步行動

拖延的根本原因常是任務不夠具體。把每件事定義成可立即執行的「下一步」，消除啟動阻力。

Concept 03
情境批次

相似性質的任務放在同一時段批次處理，減少切換成本，維持穩定的工作節律。

Concept 04
每週回顧

系統需要定期維護。每週固定一次回顧，確保所有任務都在追蹤中、沒有遺漏。

GTD 工作流程

Getting Things Done · David Allen

捕捉一切 → 釐清意義 → 組織分類 → 定期回顧 → 付諸行動

GTD 五步驟工作流
01 Capture
捕捉

把所有進入腦中的事情立刻記下來，不管大小——念頭、任務、承諾、想法，全部進收件匣。

02 Clarify
釐清

逐一處理收件匣中的每個項目：這是什麼？需要採取行動嗎？下一步是什麼？

03 Organize
組織

把釐清後的項目放進對應清單：下一步行動、等待清單、將來也許、專案清單、參考資料。

04 Reflect
回顧

每日快速瀏覽行動清單；每週完整回顧所有清單，確保系統保持最新、可信賴。

05 Engage
執行

根據當下的情境、時間、精力與優先序，選擇最適合此刻執行的任務，專注完成。

收件匣決策表

每個進入收件匣的項目，都要做出明確決定

課程單元

點擊展開內容
L01
工作現況診斷：哪裡在漏水？
Work Audit · 40 min
展開 ›

// 單元說明

在改善工作效能之前，先誠實地找出你的系統漏洞。你的任務散落在幾個地方？你有多少件事「答應要做但還沒做」？你上一次整理待辦清單是什麼時候？你的電子郵件收件匣有幾封未讀？

這些問題的答案揭示了你的工作系統中最脆弱的環節。本單元透過一份「工作效能診斷問卷」，幫你找出最需要優先修補的三個漏洞，並確立本模組學習結束後你要建立的目標系統樣貌。

// 本節重點

  • 工作系統五大漏洞：任務散落 / 沒有下一步 / 收件匣爆炸 / 缺乏回顧 / 任務混雜
  • 「開放迴路」診斷：列出所有你承諾但未完成的事
  • 計算你目前的任務分散程度（用幾個地方管理任務？）
  • 找出你最常被打斷、分心或拖延的情境
  • 設定本模組結束後的工作系統目標
本節作業 花 45 分鐘做「大腦清空」練習：把你腦中所有的開放迴路——未完成的事、承諾、擔憂、想做的事——全部寫在紙上，不加分類，只是清空。統計數量，找出最讓你有壓力感的前五項。 作業時間：45 分鐘
L02
GTD 系統：從捕捉到清零
GTD Setup · 50 min
展開 ›

// 單元說明

GTD（Getting Things Done）是由 David Allen 提出的工作管理方法論，核心思想是：把所有承諾和任務移出大腦，放進一套可信賴的外部系統，讓大腦專注於當下的工作而非記憶維護。

本單元帶你從零開始建立 GTD 系統的基礎架構：設定統一的收件匣（Inbox）、建立基本清單結構，並完成你的第一次完整收件匣清零（Inbox Zero）。這是整個工作管理系統最重要的起點。

// 本節重點

  • 建立統一捕捉入口：實體收件匣 + 數位收件匣（最多 3 個）
  • GTD 五大清單：下一步行動 / 專案 / 等待 / 將來也許 / 參考資料
  • 2 分鐘法則：可以立刻完成的事，現在就做
  • 第一次完整收件匣清零的操作流程
  • 工具選擇：Things 3 / Todoist / Notion / 紙本的各自適用情境
本節作業 選定你的任務管理工具，建立 GTD 五大清單結構。接著執行第一次收件匣清零：處理所有電子郵件、即時訊息、紙條、便利貼，將每個項目歸入對應清單或立即執行。 作業時間：2–3 小時（首次設定）
L03
任務拆解與下一步行動
Task Breakdown · 40 min
展開 ›

// 單元說明

「寫完報告」不是一個任務，它是一個專案。拖延的核心原因之一，就是任務太模糊、太龐大，讓大腦不知道從哪裡開始，於是選擇迴避。把任務拆解成具體、可立即執行的「下一步行動」，是消除啟動阻力最有效的方法。

本單元教你如何識別哪些項目是專案（需要多個步驟才能完成），如何把每個專案拆解成清晰的行動序列，以及如何用「情境標籤」讓你在不同場合能快速找到適合此刻執行的任務。

// 本節重點

  • 任務 vs. 專案的區分標準：需要超過一個步驟的都是專案
  • 下一步行動的寫法：動詞開頭 + 具體描述（「寄信給王經理確認時間」而非「聯絡王經理」）
  • 專案拆解的逆向工作法（Backward Planning）
  • 情境標籤系統：@電話 / @電腦 / @外出 / @等待 / @15分鐘內
  • 大型任務的「下一步清單」管理
本節作業 從你的任務清單中找出三個「太模糊」或「太龐大」的項目，將它們拆解成具體的下一步行動序列。為每個下一步行動加上情境標籤，確認你能夠在不需要額外思考的情況下直接開始執行。 作業時間：45 分鐘
L04
專案管理與進度追蹤
Project Tracking · 45 min
展開 ›

// 單元說明

個人的「專案」不需要複雜的甘特圖，但需要一個清晰的追蹤結構。一個專案至少需要：明確的完成定義、當前的下一步行動、以及一個定期檢視的時間點。沒有這三個要素，專案就會慢慢消失在清單的底部，成為永遠的「重要但不緊急」。

本單元介紹適合個人使用的輕量級專案管理方法，包含看板（Kanban）的簡化版本、OKR 目標設定框架在個人層面的應用，以及如何追蹤多個同時進行的專案而不感到失控。

// 本節重點

  • 個人專案清單的標準格式：名稱 / 完成標準 / 截止日 / 下一步
  • 簡化看板：待辦 → 進行中（最多 3 項）→ 等待 → 完成
  • WIP 限制（Work In Progress）：同時進行的事不超過三件
  • 個人 OKR：設定季度目標與關鍵成果
  • 每週專案回顧：確認每個活躍專案都有下一步行動
本節作業 列出你目前所有的「活躍專案」（包含工作和個人）。為每個專案填入：完成標準、預計完成日期、目前的下一步行動。找出哪些專案已經停滯（沒有下一步行動），為它們各設定一個具體的重啟步驟。 作業時間：1 小時
L05
溝通與會議效率
Communication & Meetings · 40 min
展開 ›

// 單元說明

溝通是工作中最大的時間消耗之一，但也是最少被有意識管理的部分。無效的會議、散漫的訊息往來、不清晰的請求與回覆——這些都是工作時間的隱形殺手。

本單元從兩個面向切入：首先是非同步溝通的技巧，讓你的每一封郵件和訊息都清晰、完整、讓對方一次就能回覆；其次是會議管理，包含如何判斷一個會議是否有必要，以及如何讓你主導的會議有效收尾並產出明確的行動項目。

// 本節重點

  • 非同步優先原則：能用文字解決的，不要開會
  • 清晰郵件寫作：主旨 + 背景 + 請求 + 截止日，一次到位
  • 批次處理訊息：設定固定的郵件回覆時段，而非即時回覆
  • 會議的三個前提：目標明確 / 必要出席者到場 / 有議程
  • 會議結尾必做：確認行動項目、負責人、截止日期
本節作業 本週設定兩個固定的「郵件回覆時段」（如上午 10 點和下午 4 點），其餘時間關閉郵件通知。記錄這樣的改變對工作流的影響。同時，審視本週所有會議：哪些可以取消、縮短，或改為非同步溝通？ 作業時間：1 週實驗
L06
工作收尾與日終覆盤
Daily Shutdown · 35 min
展開 ›

// 單元說明

如果說晨間例程是為一天建立良好開端，日終收尾儀式（Shutdown Ritual）則是讓工作真正「下班」的關鍵。沒有明確結束工作的儀式，大腦很難真正離開工作模式——你會在吃晚飯時想到那封還沒回的郵件，在睡前掛念著明天的會議。

Cal Newport 稱這個儀式為「計畫完整覆盤（Complete Shutdown）」。本單元設計一套可在 15–20 分鐘內完成的日終收尾流程，讓你每天以清晰、有條理的狀態結束工作，並對明天充滿把握。

// 本節重點

  • 日終收尾的五個步驟：掃描 / 更新 / 確認 / 計畫 / 宣告
  • 「計畫完整覆盤」的完整執行流程（Cal Newport 版本）
  • 如何在日終前確認所有開放迴路都有下一步
  • 設計個人的「收工咒語」：讓大腦知道工作已結束
  • 日終覆盤的三問：今天完成什麼？遇到什麼阻礙？明天最重要的一件事？
本節作業 設計你的「日終收尾流程」，寫成一份可執行的檢查清單（5–8 個步驟）。本週開始每天在固定時間執行，並在最後大聲說出你的「收工宣言」（例如：「今日工作完成。」），觀察這個儀式對你下班後放鬆程度的影響。 作業時間：30 分鐘設計 + 每日 15 分鐘執行
